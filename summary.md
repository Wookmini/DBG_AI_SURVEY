# DB Global Chip AI Tool - 아키텍처 및 보안 검토 요약

**최근 업데이트 일시:** 2026-06-12
**주제:** GitHub Pages + Google Apps Script(GAS) 배포 구조의 보안성 및 대안 검토

---

## 1. 초기 기획 아키텍처
* **프론트엔드:** GitHub Pages (현재 배포 완료 상태)
* **백엔드 (API & DB):** Google Apps Script + Google Sheets
* **특징:** 비용이 발생하지 않으며 서버 관리 불필요. 데이터가 스프레드시트에 곧바로 쌓여 실무자가 확인하고 가공하기 매우 쉬움. 
* **결론:** 기술적으로 매우 훌륭하고 구현 가능한 방식.

## 2. 사내 보안 리스크 (IT 부서 우려 사항)
현재 구조에서 **'사번'과 '성명'이라는 사내 개인정보(PII)**를 수집할 경우, 다음과 같은 치명적 보안 리스크가 발생함:
1. **API 엔드포인트 노출:** 프론트엔드 JavaScript 소스 코드에 GAS 제출 URL이 하드코딩되어, 악성 봇이나 외부인이 URL을 탈취해 가짜 데이터를 쏟아부을(Spamming) 위험.
2. **접근 통제(SSO) 부재:** GitHub Pages는 인터넷에 공개되어 있어, 사외 인원의 접속을 완벽하게 차단할 수 없음.
3. **외부망 데이터 전송:** 사내 개인정보가 퍼블릭망(GitHub)을 거쳐 외부 클라우드(Google)로 흘러가는 구조이므로 전형적인 비인가 IT 시스템(Shadow IT)으로 분류됨.

## 3. 환경 제약 및 대안 한계
* **최고의 대안 (GAS HtmlService):** GAS 자체 기능으로 배포하여 회사 구글 계정 로그인을 강제하는 방안 제안.
  * *제약사항:* **아직 모든 직원이 구글 워크스페이스 계정을 받지 못한 상태**이므로, 계정이 없는 직원이 아예 접속하지 못하는 문제가 발생하여 기각됨.
* **현실적 타협안:** GitHub Pages에 '사내 공통 암호'를 걸고, 구글 시트 백엔드에서 '임직원 명부 교차 검증'을 수행하는 이중 방어막 제안.
  * *제약사항:* 하지만 IT 보안 부서에서 **"개인정보가 외부망으로 나가는 것 자체"를 근본적 리스크로 판단**함에 따라 적용 불투명.

## 4. 최종 해결 및 우회 전략 제안
기획 의도(현업의 자동화 아이디어 취합)를 살리면서 IT/보안 부서의 우려를 원천 차단하기 위한 3가지 전략 도출:

1. **[1안] 개인정보 전면 제거 (가장 권장):** 사번과 성명 수집을 완전히 삭제하고 '부서명(드롭다운)'만 선택하게 하여 **익명 데이터**로 전환. 보안 이슈가 완전히 사라지며 즉시 배포 가능.
2. **[2안] 서버 통신 제거 (이메일 연동):** 구글 시트(GAS)로 데이터를 쏘지 않고, '제출' 클릭 시 사용자의 사내 이메일 프로그램(Outlook)이 열리며 담당자에게 메일을 쏘도록 유도 (`mailto:` 활용).
3. **[3안] 사내망 공식 호스팅:** GitHub Pages를 포기하고, IT 부서에 정식으로 요청하여 만들어둔 HTML/JS 파일들을 사내 인트라넷 서버(사내망)에 배포.

## 5. [업데이트: 2026-06-12] 보안 이슈 해결 및 GAS 연동 구현
* **결정 사항:** 보안 우려가 해소되어 최초 기획했던 **'GitHub Pages (프론트엔드) + Google Apps Script (백엔드)'** 방식을 그대로 확정 및 구현함.
* **수집 데이터:** 타임스탬프, 사번, 성명, 총점, 최종 페르소나(레벨)
* **주요 변경점 (script.js):** 
  * 사용자가 사번/성명 입력 시 전역 변수에 임시 저장.
  * 최종 결과 도출 전 `fetch` API(no-cors 모드)를 사용하여 백그라운드 비동기 통신으로 GAS 웹 앱에 데이터 `POST` 전송.
* **GAS 배포 주의사항 (트러블슈팅):** 
  * 사내 구글 워크스페이스(`dbglobalchip.com`) 환경이므로, 외부망(GitHub)에서 CORS 제약 없이 접근하려면 GAS 배포 시 **"액세스 권한이 있는 사용자: 모든 사용자(Anyone)"** 옵션이 필수적임.
  * 해당 옵션이 제한되어 있을 경우 로그인 리다이렉션(`GET`)이 발생하며 "doGet 함수를 찾을 수 없습니다" 또는 저장 실패 에러가 발생하므로 배포 권한 관리에 유의 필요.
